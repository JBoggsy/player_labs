H4 (as posed): crewborg crew has no post-task/late-game doctrine — finished/task-less crew idles solo (0.84 standing-still penalties/g vs notsus 0.01), hurting survival and getting it voted out. PHASE-1 REVISION (v82 warehouse, 227 clean eps): the mechanism as posed is REFUTED — 0/145 penalties are post-8th-task (89 all-8-done seats: zero penalties) and only 2/20 ejected crew seats idled before ejection. The idle is a MID-TASK execution freeze: 107/145 penalties sit wedged exactly one pixel outside a task-station rect (action.py ARRIVE_RADIUS=4 deadband straddles the rect edge, so the A-hold never engages: recurring stations 4/7/28/34/37), the other 38 park on dead routes (action.py:179 'a stall the mode can react to' — NormalMode never reacts). 12/173 crew games contain a frozen seat: crew win 8.3% vs 29.8% clean, ~2x game length (a frozen live seat or ghost blocks the crew task-win). Revised claim: fixing the freeze (deadband nudge + 100-tick stall escape + witness posture when task-less) eliminates the penalties and recovers those games without costing task throughput.

What's being tested

crewborg-h4:v1 = v82 base + (1) action.py complete_task deadband-wedge nudge (dead navigate mask within 24px of station center -> steer at the center, which is > ARRIVE_RADIUS inside every edge); (2) NormalMode stall escape (same spot + no task progress for 100 ticks -> block target 900 ticks, re-pick; never stand holding a dead target); (3) witness posture (live crew with no workable task shadows nearest live crewmate at 28px offset, re-picked every 200 ticks; ghosts keep the walk home). Crew-only; imposter modes untouched.

How — the instrument

Matched fresh A/B: crewborg:v82 vs crewborg-h4:v1, CREW-pinned slot 0, 2 imposters, pinned Prime top-7 field, 100 eps/arm, fired back-to-back.
target: Crewrift Prime / Competition (div_acbde92a)
subject: pinned slot 0, role=crew (game_config_overrides.slots[0])
roles: slots 1-2 imposter, 3-7 crew (always-2-imposter pref)
opponents (identical both arms, slot -1 rotate): crewrift-prime-crewborg-aaln-hunter-relhalpha:v1, jordan-crewborg-aaln:v1, crewborg-mv:v1, notsus:v132, daveey-prime-notsus:v2, forgeling-focusfire:v6, crewborg-aaln:v18

measure (subject seat only): standing-still penalties/g, tasks completed/g, survive-to-end, voted-out-as-crew, crew win, mean score

Predictions — the falsifiability contract

If TRUE — confirms
Candidate standing-still penalties/g collapse to notsus-level (< 0.15/g, i.e. >80% reduction vs the baseline arm) AND tasks/g does not drop (>= baseline - 0.15). Directionally (underpowered at 100 eps): fewer 10k+-tick marathon losses, crew win/survival flat-to-up.
If FALSE — refutes
Candidate penalties/g stay >= 0.4/g (escape/nudge failed to clear the wedge), OR tasks/g drops > 0.15 / crew win drops significantly (the escape abandons workable tasks or the shadow posture displaces task work) — the fix is wrong or harmful.

Decision rule (committed before running)

Read as: PRIMARY (mechanism): candidate ss-penalties/g < 0.15 with baseline >= 0.5 in the same window => CONFIRMED; candidate >= 0.4 => REFUTED. GUARDS: tasks/g drop <= 0.15 and no significant crew-win regression (compare.py 'regressed'), else the fix is rejected regardless. Crew win/survival deltas are reported with CIs but NOT gating (1-2pp effects are unresolvable at 100 eps). Committed before firing the requests.

Confounds controlled

  • Field drift — both arms pin the identical 7-opponent roster by exact name:vN and fire back-to-back; pool selectors are not used.
  • Role mix — subject seat is role-pinned crew in both arms; per-seat bias cancelled by rotating all opponents (slot -1).
  • Ops failures hit arms asymmetrically — connect/disconnect-timeout episodes are dropped from both arms before comparing.
  • Env skew — candidate uploaded with CREWBORG_METRICS=1 + CREWBORG_TRACE_GROUPS=all (observability only), matching the v82 upload recipe.
  • Baseline tree — baseline is crewborg:v82 (built from main f3bcd17, the same commit the candidate branch forked from), so only the H4 diff differs.
  • Crew-pinned masking — a crew-pinned A/B can mask natural-role effects; acceptable here because the mechanism under test is crew-only and imposter code is untouched (confirm in natural roles before shipping).

Verdict

confirmedA/B (97 vs 100 clean eps, crew-pinned, identical pinned field): standing-still penalties/g 0.454 -> 0.010 (44 total -> 1; penalty-split binomial p=6.8e-13); games-with-penalties 7/97 -> 1/100 (Fisher p=.033); frozen games (>=2 penalties) 4/97 -> 0/100 (p=.057); 40/44 baseline penalties sat wedged at a task-rect edge, candidate's single penalty was off-station. Voted-out-as-crew 11.3% -> 2.0% (p=.0094) — a significant, unpredicted improvement. Guards: tasks/g 6.62 -> 6.43 (point drop 0.19 vs the 0.15 gate, but p=.53 — noise; both straddle the league 6.49); crew win 30.9% -> 25.0% (p=.43, noise); survival 29.9% -> 34.0% (p=.55, noise); marathons(>10k ticks) 22 -> 17. LETTER-OF-THE-RULE NOTE: the baseline-reproduction clause asked >=0.5 ss/g and the arm produced 0.454 (91% of gate), and the tasks/g point drop exceeded 0.15 while being statistically flat — by strict conjunction the pre-committed rule reads inconclusive; by its primary contrast (candidate <0.15 achieved at 0.010, refutation clause >=0.4 not met, pathology clearly reproduced in-arm) the mechanism fix is confirmed. H4 AS ORIGINALLY POSED (post-task doctrine gap) remains REFUTED by Phase 1: 0/145 league penalties were post-8th-task; the idle was a mid-task deadband wedge + unowned no-route hold.
