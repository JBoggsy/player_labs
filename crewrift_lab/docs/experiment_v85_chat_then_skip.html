How — the instrument

Per-meeting engine attribution from telemetry event stream, joined with results.json ground-truth vote outcomes.
Per v85 crew meeting: chats (accusation = names another color + accusatory keyword), meeting_llm_decision actions/vote_targets, meeting_llm_fallback (llm_call_failed + error class), meeting_tentative_vote, meeting_vote_selected engine via the code's own reason strings ('meeting deadline: auto-submit tentative vote' = AUTO_SUBMIT, 'LLM fallback after deadline' = LLM_FAIL_DEADLINE, 'deterministic meeting vote' = DET, else LLM_SUBMIT), belief-time geometry (believed remaining at vote, max tick jump), cumulative meeting_llm.latency_ms. Ground truth: results.json vote_players / vote_skip / vote_timeout per seat.

Predictions — the falsifiability contract

If TRUE — confirms
Among crew meetings with ≥1 accusation-style chat that end without a player vote, the modal pattern (≥50%, ≥2× the next mechanism) is: successful chat decisions with no vote_target + ≥1 llm_call_failed + final vote from the deadline path with no tentative set ⇒ fallback-gate skip.
If FALSE — refutes
Those meetings are instead dominated by: the LLM explicitly choosing skip / withholding targets with no failures (M2), deadline/clock geometry where no vote fires or lands (M3), or deterministic share_read chat with a separately-gated vote (M4).

Decision rule (committed before running)

Read as: M1 (fail-then-gate-skip) ≥ 50% of confident-chat-no-player-vote crew meetings AND ≥ 2× the next largest mechanism ⇒ CONFIRMED. M1 < 30% or another single mechanism ≥ M1 ⇒ REFUTED. Otherwise inconclusive. Committed before aggregation.

Confounds controlled

  • Player-name collision (v82/v84 also 'James Boggs') — controlled: seat by policy_version_id position, role cross-checked per episode (0/175 mismatches).
  • Role mixing — controlled: crew (crewmate) meetings only for the primary read; imposter reported separately.
  • Meetings ended by GameOver/Playing or truncated telemetry — excluded from vote-outcome attribution (5 of 168 accusation meetings).
  • Accusation classifier is a text proxy — sensitivity via a strict subset (sus/vent/imposter/liar/guilty/vote keywords); results unchanged in direction.
  • 429 clustering in league time — reported as error-class distribution, not assumed uniform.

Verdict

confirmed163 crew meetings with accusation chat (of 230 crew meetings, 112 episodes). 102/163 (63%) ended with no player vote selected. Mechanism split: M1 fail-then-gate-skip 60/102 (59%) — meets the ≥50% and ≥2×-next rule (next: M3 clock/no-vote 27, M2 LLM chose skip 10, M2b withheld 5). Root numbers: send_chat decisions carry vote_target in 1/1566 cases; only 82/1657 (5%) of successful LLM decisions are vote-typed; 35% of calls fail (800 × Bedrock 429 'Too many tokens per day', 86 timeouts). COMPOUNDING LAYER the hypothesis missed: every synchronous LLM call blocks the game loop ~3s (bridge.loop_gap_ms ≈ call latency); skip-meetings average 29s of blocking in a 50s meeting, so the belief clock lags reality by up to ~670 ticks — 27 meetings never reached their believed deadline before the real meeting ended, and of votes that WERE selected, 21/77 player votes and 34/78 skips never landed (60/230 crew meetings = 26% ended vote_timeout). VOTE_TIMER 240→1200 (v84) is not the bug; belief-time lag is.
